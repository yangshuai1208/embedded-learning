1.指针的本质（地址、间接访问）
指针即地址:指针变量存储的是内存单元的地址，通过地址可以定位到数据在内存中的位置。
间接访问：通过解引用（*运算符）访问指针指向的内存单元，实现对数据的读写。
指针类型：指针类型决定了从地址开始读取多少字节，以及如何解释这些字节，。
指针变量本身也有地址，就是二级指针。

2.指针运算与数组的关系
数组名一般退化为指针，数组名为首元素地址，例外（sizeof(arr)和(&arr))。
下标访问的本质：指针算术运算以元素大小为移动单位。
指针加减整数：常用语遍历数组，p + n 移动 n * sizeof(*p) 字节。

3.动态内存分配（malloc/free）的注意事项
分配函数：malloc(sizeof)分配size字节，返回void*，失败返回NULL，ccalloc额外清零，realloc调整大小。
必须检查返回值：使用前判断p!=NULL，否则解引用空指针导致奔溃。
配对释放：每个malloc成功的内存最终都要通过free释放，否则造成内存泄漏。
释放后置NULL：free(p)；p=NULLL;可避免悬挂指针。
重复释放：对已释放的内存再次调用 free 是未定义行为。
越界访问：写入分配区域之外会破坏堆管理结构，导致程序难以调试的错误。

4.结构体的定义和使用，结构体指针
定义struct 标签{成员列表};通常搭配typedef简化类型名。
变量声明:struct 标签 变量；或类型别名 变量；
成员访问:普通变量用.运算符，
指针变量用->运算符。
内存对齐：结构体成员按自然边界对齐，整体大小为最大成员对齐值的整数倍（可通过 #pragma pack 调整）。
自引用：结构体内部可以包含指向自身类型的指针，用于构建链表、树等数据结构。
作为函数参数：通常传递结构体指针以避免拷贝开销，并允许函数修改原结构体。

5.函数指针的用途（回调函数）
声明：返回类型 (*指针名)(参数列表); 例如 int (*p)(int, int);
赋值与调用：p = &add; 或 p = add;（函数名自动转指针）；调用 p(2,3) 或 (*p)(2,3)。
回调函数：将函数指针作为参数传递给另一个函数，由后者在特定时机调用，实现解耦和灵活扩展。典型应用：
标准库 qsort：通过比较函数指针定制排序规则。
事件驱动编程：注册回调函数，当事件发生时被调用。
遍历算法：将处理逻辑作为回调传入，实现算法与数据分离。
函数指针数组：用于实现状态机、命令分发等，提高代码模块化程度。
