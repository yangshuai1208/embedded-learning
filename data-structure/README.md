一、链表的创建、插入、删除、反转思路
1. 创建（头指针 + 节点）
思路：定义节点结构体（数据域 + 指针域），维护一个头指针（初始为NULL）。每次创建新节点时动态分配内存（或从静态池获取），初始化数据域和next指针为NULL，然后根据需求插入链表。
嵌入式注意：如果使用malloc，必须检查返回值；裸机环境常用静态节点池，避免碎片。
2. 插入（头插法、尾插法、中间插入）
头插法：新节点next指向当前头，然后更新头指针。O(1)
尾插法：先遍历到尾部（while(p->next) p = p->next;），再将尾节点next指向新节点。O(n)
中间插入：找到插入位置的前驱节点，新节点next指向后继，前驱next指向新节点。
3. 删除
按值/按位置删除：需要保存前驱指针（或使用二级指针简化）。如果是头节点，直接更新头指针；否则前驱的next指向被删节点的next，然后free（或归还到池中）。
注意：删除节点前，务必先保存其next指针，防止丢失链表。
4. 反转（迭代法最常用）
思路：用三个指针prev、curr、next遍历。prev初始NULL，curr指向头。循环内先保存next = curr->next，然后将curr->next指向prev，再移动prev = curr，curr = next。最后头指针指向prev。
递归法：但嵌入式慎用递归，防止栈溢出。
常见坑
插入/删除时忘记更新头指针。
遍历时使用了已被释放的节点（悬垂指针）。
在中断与任务共享链表时未加保护（需关中断或使用互斥锁）。

二.栈和队列的特点
栈	先进后出（LIFO）。只在栈顶操作。	函数调用、中断嵌套、表达式求值、括号匹配、回溯算法。
队列	先进先出（FIFO）。队头出队，队尾入队。	消息缓冲、任务调度、数据流处理（如串口环形缓冲）。
嵌入式视角：
栈通常由硬件SP指针管理（如任务栈），程序员需确保栈空间足够，避免溢出。
队列常用环形缓冲（数组实现），适合中断与主循环的数据传递，无需动态内存。

三、循环队列如何解决假溢出
假溢出现象
普通顺序队列（数组实现）在多次入队、出队后，队尾指针rear可能移动到数组末尾，而数组前部还有空闲位置，但无法继续入队（看似已满）。
循环队列原理
将数组逻辑上视为环形，使用取模运算使rear和head在数组范围内循环。
关键操作：
// 入队
rear = (rear + 1) % MAX_SIZE;
buffer[rear] = data;
// 出队
head = (head + 1) % MAX_SIZE;
data = buffer[head];
判空/判满常用两种方法：
牺牲一个存储单元：(rear + 1) % MAX_SIZE == head 为满，head == rear 为空。
增加count成员记录元素个数：更直观，但需要原子操作。
为什么能解决假溢出
因为当rear到达数组末端时，通过取模操作可以回到数组起始位置，从而利用之前出队释放的空闲空间，实现循环复用。
嵌入式坑点
索引类型要足够（uint16_t或uint32_t），避免溢出。
在中断中操作时，务必保证head/rear的读写在临界段内完成，防止数据错乱。

四、坑或理解难点
链表的二级指针
刚开始时总搞不清楚为什么函数内修改头指针要传Node**。后来明白：如果要在函数内改变头指针本身（如删除头节点），必须传入头指针的地址（二级指针），否则形参只是头指针的副本，修改无法传出。
环形队列“满”的判断
使用“牺牲一个单元”法时，总是忘记数组实际可用容量是MAX_SIZE-1。容易在代码中错误地写if(rear == head)判满，导致无法区分空和满。后改用count变量，虽然多占一点内存，但逻辑更清晰，也适合多生产者消费者场景。
结构体内存对齐
在STM32中，定义链表节点时如果不加__packed，结构体可能占用超过成员总和的字节数，导致链表节点内存浪费。但在某些总线传输（如USB）中，又需要自然对齐以保证访问速度。这需要根据实际硬件和编译器特性权衡。
在中断中遍历链表
有一次在串口中断中遍历一个全局链表，结果主循环正在修改链表，导致中断里访问了已释放的节点，系统hard fault。后来学到：中断与任务共享数据结构时，必须禁止中断或使用无锁机制（如双缓冲）。
栈溢出检测
调试RTOS任务时，某个任务莫名崩溃，后来发现是任务栈太小，局部变量溢出破坏了TCB。通过填充栈区特殊字（如0xDEADBEEF）并周期性检查，终于定位问题。从此深刻理解了“栈”的边界管理。
链表反转的边界条件
第一次手写反转时，忘了处理空链表和单节点链表的情况，导致反转后头指针为NULL。后来习惯在写循环前先判断特殊情况。
排序算法的稳定性
在做多优先级任务调度时，想用快速排序按优先级排序，结果导致相同优先级的任务顺序改变，影响了预期顺序（比如FIFO要求）。后来改用稳定排序（插入排序）解决了问题。
